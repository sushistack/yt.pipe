## 2. What Went Well

### 2.1 Pattern Reuse
Existing store patterns (`project.go`, `feedback.go`) were directly applicable. No new architectural patterns needed — consistent error wrapping, time parsing, transaction handling, and `NotFoundError` usage throughout.

### 2.2 Migration Auto-Discovery
The `go:embed migrations/*.sql` pattern meant adding `003_templates.sql` was sufficient. Zero changes to `store.go` migration engine. Naming convention (`NNN_description.sql`) handled ordering automatically.

### 2.3 Embedded Default Templates
Used `go:embed default_templates/*.md` to bundle proven prompts from the existing `templates/` directory into the service binary. Deployment stays simple — no external file dependencies.

### 2.4 Test Coverage
51 test functions across domain (3), store (22), service (19+2), and CLI (9) layers. Key behaviors verified: CRUD operations, 10-version auto-pruning, rollback creating new versions, default template protection, project override resolution priority, and idempotent installation.

## 3. Challenges & Growth Areas

### 3.1 Cobra initConfig Conflict
CLI tests using `rootCmd.Execute()` triggered `initConfig()` which overwrote the test `appConfig`. Resolved by testing service layer directly and testing CLI helper functions (e.g., `installDefaultTemplates()`, `templateCategoryFromString()`). Full cobra integration test pattern remains weak.

### 3.2 Architecture Document vs Reality — Migration Numbering
Architecture document specified `002_templates.sql`, but `002_feedback.sql` already existed from Epic 6. Correct numbering is `003`. Downstream epics (14→004, 15→005, 17→006, 16→007) need corresponding shifts. Not a code issue — documentation discrepancy only.

### 3.3 Story Spec Files Not Created
Only Story 13-1 had a formal implementation artifact (`13-1-prompt-template-domain-model-database-migration.md`). Stories 13-2 through 13-5 were implemented directly from the epic definition without running the `create-story` workflow. This worked well for a single-session approach but bypasses the BMAD story creation process.

### 3.4 Caption Default Template Unverified
The `scenario.md`, `image.md`, and `tts.md` default templates were copied from existing proven prompt files. The `caption.md` template was newly authored and has not been validated through the actual pipeline.
